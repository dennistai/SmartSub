# 語音／字幕多語處理規則（中文・泰文・英文）

> **用途**：可跨專案重用的「語言處理規則」規格。與任何特定程式碼、框架無關。
> 描述「輸入語音／字幕 → 最終統一為繁體中文」的判斷與轉換規則。
> 實作者可自行選擇工具（OpenCC、Whisper、Google 翻譯、LLM…）落地。

---

## 0. 總則

- **最終輸出語言一律為「繁體中文（台灣）」。**
- 語音辨識模型**無法辨識說話者地區或口音**：同一段中文，輸出是簡體或繁體，取決於**模型本身特性**，而非說話者是台灣人或大陸人。因此不可用「說話者地區」推斷字形或用詞。
- 規則分兩層，**互相獨立**：
  - **字形**（繁體 vs 簡體字元）
  - **用詞**（台灣用語 vs 大陸用語）
  兩層要分開判斷、分開開關，不可綁在一起。

---

## 1. 中文字形規則

**目標：中文最終一律繁體，且盡量保留模型原生輸出、避免非必要轉換。**

| 偵測到的原生字形 | 動作 |
|---|---|
| 繁體 | **保留原生**，不做任何轉換 |
| 簡體 | 轉成繁體（字形層） |
| 無法判斷（純數字／標點／外文） | 不動 |

規則細節：

1. 先**偵測轉寫結果的實際字形**（不要假設某模型固定輸出哪種字形；以實際內容為準）。
2. 偵測為繁體 → 直接保留，**跳過轉換工具**（避免對已正確的繁體做一對多誤轉）。
3. 偵測為簡體 → 轉繁體。
4. **永不**將中文轉成簡體（不論來源如何，簡體只會是「待轉繁」的中間狀態）。
5. 判斷傾向繁體：偵測到「簡體訊號」需嚴格大於「繁體訊號」才判為簡體；相等或中性一律當繁體處理。

---

## 2. 中文用詞規則（獨立開關，預設關閉）

**背景**：語音模型分不出台灣人／大陸人。台灣人講話被辨識成簡體字時，**用詞其實仍是台灣用詞，只是字形被寫成簡體**。此時只需字形轉換即可；若強行做「用詞轉換」，反而會把已正確的台灣用詞誤當大陸用詞去「校正」。

| 用詞轉換開關 | 動作 | 適用情境 |
|---|---|---|
| **關（預設）** | 只轉**字形**，保留原用詞 | 台灣講者被誤判成簡體字時 |
| **開** | 字形＋**台灣用詞轉換**（例：信息→資訊、軟件→軟體、自行車→腳踏車） | 內容確實是大陸用詞（與大陸方開會）時 |

規則細節：

1. 用詞轉換**只在「簡→繁」轉換路徑上**才有意義；原生繁體被保留時（規則 1.2）不套用。
2. 預設**關閉**，需人工按會議情境決定是否開啟。
3. 用詞轉換僅限「詞典級」替換能力；超出詞典的語意潤飾屬翻譯／校對層，另議。

> **參考實作對照（OpenCC）**：字形轉換用 `s2tw`（簡→台灣繁體，只轉字形）；用詞轉換開啟時用 `s2twp`（簡→台灣繁體＋台灣用詞）。作終端顯示請用 `tw`／`twp`，勿用中繼標準形 `t`。

## 3. 多語混合規則（中文・泰文・英文）

**情境**：一場會議／一段音檔常同時出現三種語言：

1. 台灣的**繁體中文**
2. 泰國的**泰文**
3. **英文**（雙方懂英文者常以英文溝通，或英文充當翻譯橋樑）

**目標**：偵測各語言，全部統一輸出為**繁體中文**。

需要**逐段（per-segment）語言偵測**，再依語言分流：

| 偵測到的語言 | 動作 |
|---|---|
| 中文 | 套用規則 1（字形）＋規則 2（用詞）→ 繁體中文 |
| 泰文 | 翻譯成繁體中文 |
| 英文 | 翻譯成繁體中文 |

---

## 4. 泰文翻譯的樞紐規則（Pivot）

**經驗**：泰文**直接**翻成繁體中文，語意常常出錯、品質差（尤以 Google 翻譯明顯）。先經英文中轉，語意較準確。

規則：

1. 優先嘗試：**泰文 → 繁體中文**（直接）。
2. 若直接翻譯品質不佳／有困難：改走**樞紐翻譯**：
   ```
   泰文 → 英文 → 繁體中文
   ```
3. 英文中間結果只作橋接，**最終交付仍為繁體中文**。

> **實作提示**：
> - 若使用**傳統翻譯 API**（如 Google 翻譯），建議固定採用「泰→英→繁」兩段式樞紐，品質較穩。
> - 若使用**LLM 翻譯**（如具備多語能力的大型語言模型），通常可一步處理混合語言輸入並直接輸出繁體中文，樞紐可視情況省略；仍建議對泰文品質做抽查。

---

## 5. 規則優先序摘要（可直接當實作 checklist）

```
對每個語音／字幕片段：
  1. 偵測語言
  2. if 中文:
       a. 偵測字形
       b. 繁體 → 保留；簡體 → 轉繁（字形層 s2tw）
       c. 用詞轉換開關為「開」時 → 套用台灣用詞（s2twp）
       d. 永不轉簡
  3. elif 泰文:
       a. 嘗試 泰→繁
       b. 品質不佳 → 泰→英→繁（樞紐）
  4. elif 英文:
       a. 英→繁
  5. else（其他語言）:
       a. 依需要翻成繁，或保留（依專案政策）
最終輸出：一律繁體中文（台灣）
```

## 6. 設計決策記錄（Why）

- **字形與用詞拆兩層／兩開關**：模型分不出地區口音，台灣講者被轉簡體時用詞仍是台灣用詞；用詞轉換獨立、預設關，避免誤校正。
- **偵測而非模型白名單**：以實際輸出字形判斷，與模型解耦、免維護模型清單、未來新模型自動適用。
- **繁體優先、永不轉簡**：最終語言統一為繁體是硬性目標，簡體僅為中間態。
- **泰文走英文樞紐**：實測直翻品質差，英文中轉語意較準。
